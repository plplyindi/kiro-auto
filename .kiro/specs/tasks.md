# 技术文章自动分析任务

请按照以下步骤分析技术文章，帮助评估学习价值和实现方案。

## 任务说明

每次运行分析 **最多 3 篇** 技术文章。

---

## 第 1 步：检查待分析文章

1. 查看 `articles/` 目录中的所有文章文件（支持 .md、.txt、.html 格式）
2. 排除 README.md 文件
3. 按文件名排序，选择前 3 篇文章
4. 如果没有文章，创建一个 `no-articles.txt` 文件说明"暂无待分析文章"并结束

**支持的格式：**
- `.md` (Markdown) - 推荐
- `.txt` (纯文本)
- `.html` (HTML 网页) - 自动提取文本内容

---

## 第 2 步：深度分析每篇技术文章

对选中的每篇技术文章，执行以下专业分析：

### A. 基础信息提取

1. **文章标题**
2. **主要技术栈/框架/工具**
3. **技术领域**（如：前端、后端、AI/ML、DevOps、数据库等）
4. **难度级别**（入门/中级/高级/专家）

---

### B. 学习价值评估 ⭐ 核心分析

#### 1. 是否值得学习？（给出明确建议）

评估维度：
- **技术前沿性**：是否是当前热门或未来趋势技术？
- **实用性**：在实际项目中的应用广度
- **成熟度**：技术生态是否完善（文档、社区、工具链）
- **就业价值**：掌握该技术对职业发展的帮助

**输出格式：**
```
学习建议：强烈推荐/推荐/选择性学习/不推荐
理由：（详细说明）
适合人群：（如：有React基础的前端开发者）
```

#### 2. 实现需要多少时间？

分阶段估算：
- **理解概念**：X 小时（阅读文章、理解原理）
- **动手实践**：X 天（跟随教程、完成 Demo）
- **项目应用**：X 周（在实际项目中应用）
- **深入掌握**：X 个月（掌握高级特性、最佳实践）

**前置技能要求：**
- 需要掌握哪些技术？
- 是否需要相关经验？

---

### C. 实现方案分析

#### 1. 核心技术要点（3-5 个关键点）

示例：
- 使用 WebSocket 实现实时通信
- 采用 Redis 缓存提升性能
- 通过 Docker 容器化部署

#### 2. 实现步骤概述

将文章中的技术实现拆解为清晰的步骤：

**步骤 1**: 环境准备
- 安装 xxx
- 配置 xxx

**步骤 2**: 核心功能实现
- ...

**步骤 3**: 优化和部署
- ...

#### 3. 技术难点识别

- 列出 2-3 个实现中可能遇到的难点
- 给出解决思路或参考资源

#### 4. 依赖和工具

- 编程语言：
- 框架/库：
- 开发工具：
- 第三方服务（如果有）：

---

### D. 技术扩展应用 ⭐ 核心分析

#### 这个技术可以在哪些场景应用？

**分析维度：**

1. **直接应用场景**（文章提到的场景）
   - 场景 1：...
   - 场景 2：...

2. **扩展应用场景**（举一反三）
   - 场景 1：如何应用？有什么优势？
   - 场景 2：如何应用？有什么优势？
   - 场景 3：如何应用？有什么优势？

**举例说明：**
```
如果文章讲的是"用 Redis 实现分布式锁"

直接场景：
- 分布式系统中防止重复操作

扩展场景：
1. 秒杀系统：控制库存并发访问，防止超卖
2. 定时任务：确保定时任务在集群中只执行一次
3. 订单处理：防止用户重复提交订单
4. 资源预订：酒店/机票预订系统的座位锁定
```

3. **技术组合应用**
   - 可以和哪些其他技术结合？
   - 组合后能解决什么问题？

4. **行业应用**
   - 电商、金融、教育、医疗等行业如何使用？

---

### E. 综合评分与建议

1. **技术价值评分**（1-10 分）
   - 创新性：X/10
   - 实用性：X/10
   - 可维护性：X/10
   - 性能表现：X/10
   - **总分**：XX/40

2. **学习建议**
   - 立即学习 / 列入计划 / 先观望

3. **潜在风险**（如果有）
   - 技术限制
   - 兼容性问题
   - 维护成本

4. **参考资源**
   - 官方文档
   - 相关教程
   - 开源项目

---

### 输出格式：

为每篇文章创建详细分析报告，保存到 `results/details/` 目录：

**文件名**：`analysis-{原文章名}-{日期时间}.md`

**内容结构**：使用清晰的 Markdown 格式，包含以上所有分析维度

---

## 第 3 步：生成汇总报告

更新 `results/summary.md`，包含：

### 本次分析概览
- 分析时间：YYYY-MM-DD HH:MM
- 分析文章数：X 篇
- 文章列表及主要技术

### 技术趋势分析
- 本次涉及的技术领域分布
- 推荐学习的技术（按优先级）
- 值得关注的技术方向

### 实现时间统计
- 快速上手（< 1周）：X 篇
- 中等投入（1-4周）：X 篇
- 长期学习（> 1月）：X 篇

### 应用场景汇总
- 列出所有分析出的应用场景
- 按行业或场景类型分类

---

## 第 4 步：归档已分析文章

将已分析的文章从 `articles/` 移动到 `analyzed/` 目录

---

## 第 5 步：创建处理记录

创建 `processing-log.txt`，记录：
- 处理时间
- 处理的文章列表和主要技术
- 剩余待处理文章数量

---

## 注意事项

1. **如果文章包含 URL**：尝试访问链接获取完整内容
2. **代码示例**：如果文章有代码，简要说明代码的作用
3. **保持客观**：基于技术价值和实用性评估，避免主观偏见
4. **具体化建议**：给出可操作的学习路径和实现方案

---

**开始深度分析技术文章！**
